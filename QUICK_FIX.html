üö® QUICK FIX: Database "dcms" Does Not Exist

Problem

Your application can't connect because the PostgreSQL database "dcms" was never created or was deleted.

Solution (30 seconds)

Run this one command:

./create_database.sh

This script will:

  1. ‚úÖ Create the "dcms" database
  2. ‚úÖ Restart the backend to initialize tables
  3. ‚úÖ Verify everything is working

Manual Steps (if script doesn't work)

# 1. Create the database
docker-compose exec db psql -U postgres -c "CREATE DATABASE dcms;"

# 2. Restart backend to create tables
docker-compose restart backend

# 3. Wait a few seconds
sleep 5

# 4. Verify tables were created
docker-compose exec db psql -U postgres -d dcms -c "\dt"

After Database is Created

Restore from NetBox:

# Option 1: Use the Web UI
# Go to http://localhost:3000/netbox
# Click "Full Sync from NetBox"

# Option 2: Use the API
curl -X POST http://localhost:8000/api/v1/netbox/sync/pull

Or restore from backup (if you have one):

curl -o backup.json http://localhost:8000/api/v1/backup/export
docker-compose exec backend python backup_cli.py import backup.json

Prevent This in the Future

The database should be automatically created by docker-compose. Check your docker-compose.yml:

services:
  db:
    image: postgres:15
    environment:
      POSTGRES_DB: dcms        # ‚Üê This creates the database
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres

If POSTGRES_DB: dcms is missing, add it and run:

docker-compose down -v  # WARNING: Deletes all data
docker-compose up -d    # Recreates with new database
